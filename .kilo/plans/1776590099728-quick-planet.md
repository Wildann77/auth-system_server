# Premium Upgrade Feature Implementation Plan

## Overview
Implement a premium upgrade system where users can purchase premium status through payment, automatically gaining access to exclusive features upon successful transaction completion.

## Database Schema Changes

### User Model Updates
- Add `isPremium: Boolean` field (default: false)
- Add `premiumUntil: DateTime?` field for subscription expiry (optional for future extensions)
- Update indexes if needed for premium user queries

### Order Model Updates
- Add `orderType: Enum` field with values: GENERAL | PREMIUM_UPGRADE
- Default to GENERAL for existing orders
- This distinguishes premium upgrades from regular purchases

## Payment System Updates

### Checkout Endpoint Modifications
- Extend `/api/v1/payment/checkout` to accept `orderType` parameter
- For PREMIUM_UPGRADE orders:
  - Hardcode premium price on server-side (prevent client manipulation)
  - Validate user is not already premium
  - Store userId in order metadata for webhook processing
- Return appropriate payment URLs/tokens as before

### Webhook Handler Updates
- Modify both Midtrans and Stripe webhook handlers
- On successful payment (`SUCCESS` status):
  - Check `orderType`
  - If `PREMIUM_UPGRADE`: Find associated user and set `isPremium: true`
  - Ensure idempotency: only upgrade once per successful order
- Update order status as usual

## Authentication & Access Control

### New Middleware
- Create `requirePremium` middleware
- Check `req.user.isPremium === true`
- Return 403 Forbidden for non-premium users

### Protected Endpoints
- Create `/api/v1/content/exclusive` endpoint (example)
- Apply `requirePremium` middleware
- Add more premium-only features as needed

## Security & Token Management

### Token Handling
- On successful premium upgrade, increment user's `tokenVersion`
- This forces refresh token rotation, invalidating existing sessions
- Users must login again to get new access token with premium claims

### Idempotency
- Webhook processing ensures upgrade only happens once per order
- Check order status before processing upgrade
- Log upgrade events for audit trail

## Implementation Steps

1. Update Prisma schema with new fields
2. Run database migration
3. Modify checkout endpoint for premium orders
4. Update webhook handlers for auto-upgrade logic
5. Create requirePremium middleware
6. Add premium-protected endpoints
7. Update types and interfaces
8. Test payment flow and access control

## Testing Strategy

- Unit tests for middleware and schema validation
- Integration tests for payment flow
- Webhook simulation tests
- Access control tests for premium features
- Idempotency verification

## Rollback Plan
- If issues arise, can revert premium flags in database
- Monitor webhook logs for failed upgrades
